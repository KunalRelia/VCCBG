# Lean4 Formalization of the VCCBG paper
This repository is aimed at formalizing the algorithmic results of the vertex cover problem on cubic bridgeless graphs (VCCBG) [paper](https://kunalrelia.github.io/img/VCCBG.pdf) using Lean4. Currently, we are focusing on the formalization of the proof of correctness of the alternative algorithm presented in Section C of the paper, which proves that VCCBG $\in$ P.

_Disclaimer: The Lean code was generated and refined over multiple iterations using Claude Sonnet 4.6 (using the Low or Max thinking modes). Gemini, ChatGPT, and Claude were used for debugging._

## Build the Lean files

To build the Lean files of this project, you need to have a working version of Lean installed on your machine.
See [the installation instructions](https://lean-lang.org/install/).

Next, please clone this repository. Then, follow these steps:

```
% cd VCCBG/
% lake exe cache get (or lake exe cache get! for complete download)
% lake build
```

## An AI-generated summary of the Lean files
_(We uploaded all the current Lean files to Gemini and asked "so can you pls summarize what these files do using this 'post123' file as the final stop?". The ordering of the following response is edited at one place.)_

Taken together, this collection of Lean4 files formalizes a complete, rigorous verification pipeline for a local search algorithm designed to find a Minimum Vertex Cover on Cubic Bridgeless Graphs.

The files trace a beautiful logical arc: they start with a pure graph-theoretic equivalence, define how a single local search step behaves, build up the mechanics of a specialized coloring subroutine, and finally arrive at the "post123" file, which acts as the ultimate capstone proving that the algorithm’s search is 100% complete.

Here is the step-by-step summary of how these files assemble that final proof, grouped by their roles.

### Phase 1: The Theoretical Foundation
#### `Thm12.lean` (Theorem 12)
This is the mathematical bedrock of the entire project. It proves a pure, bidirectional structural theorem: a vertex cover S can be strictly shrunk if and only if the graph contains an "S-diminishing bipartite graph." This means the algorithm doesn't need to look for abstract, arbitrary improvements; it just needs a way to hunt down these specific bipartite structures.

### Phase 2: Algorithm Integrity & Main Loops
#### `Thm13Lemma9.lean` (Lemma 9)
Proves the high-level correctness and termination conditions for the parent wrapper (Algorithm A). It ensures that if a minimum vertex cover of a certain size exists, the algorithm will not prematurely exit with a "No" and will safely terminate with an optimal configuration.

### Phase 3: The Micro-Mechanics of the Search Subroutine (Algorithm C)
#### `Thm13Lemma11.lean` (Lemma 11)
Focuses on Algorithm B (the main optimization loop). It establishes a crucial loop invariant: no matter how many times the algorithm modifies, swaps, or updates its current set of vertices, the set S is guaranteed to remain a valid vertex cover at every single execution step.

To find the diminishing bipartite graphs promised by Theorem 12, the algorithm runs a coloring subroutine (Algorithm C) that "seeds" a vertex and radiates outward using strict forcing rules (V_1 through V_4).

####  `Thm13Lemma12_2b.lean` & `Thm13Lemma12_overall.lean` (Lemma 12)
These files handle the coloring phase. They prove that when you plant a "blue" seed at a vertex, the forcing rules yield a unique coloring along active reachability paths. File 2b specifically proves how structural obstacles—like odd cycles—reconcile conflicts by forcing a "black" (neutralizer) vertex.
#### `Thm13Lemma13.lean` (Lemma 13)
Proves that the resulting blue and red vertices generated by this coloring process are well-behaved: they never clash, meaning the induced subgraph of colored vertices is successfully bipartite.
#### `Thm13Lemma14.lean` (Lemma 14)
Elevates the bipartite graph to an S-alternating bipartite graph by proving that all blue vertices are safely trapped inside the current vertex cover S, while all red vertices are strictly outside of it.
#### `Thm13Lemma15.lean` (Lemma 15)
Proves that if this alternating graph contains more blue vertices than red vertices (∣U∣>∣W∣), executing a "flip" (removing the blue vertices from the cover and adding the red vertices instead) is mathematically guaranteed to output a strictly smaller, valid vertex cover.
### Phase 4: The Final Destination (The Lemma 16 Multi-Part)
Now that the per-seed mechanics are fully verified, the codebase has to prove that iterating this process over all possible seeds will never miss a solution.
#### `Thm13Lemma16_123.lean` (Lemma 16, Points 1–3)
This file collects the individual wins from Lemmas 12–15. Utilizing Petersen's Theorem (which guarantees a perfect matching in cubic bridgeless graphs), it proves that every single vertex in the graph will be evaluated as a seed, and establishes the local "per-seed logical chain" of unique coloring ⟹ unique alternating graph ⟹ potential reduction.
#### `Thm13Lemma16_post123.lean` (The Final Stop)
This is where the entire repository reaches its climax. This file handles the closing, global contrapositive argument of Lemma 16.
It asks the critical question: What if Algorithm B checks every single seed vertex in the graph, and absolutely none of them yield a smaller vertex cover? By sewing together the seed-coverage maps, the components captured by the boundaries, and Theorem 12, `Thm13Lemma16_post123.lean` delivers the final, definitive proof: if no local improvement is found anywhere, then no S-diminishing bipartite graph exists anywhere in the entire graph. Therefore, the algorithm's failure to find an improvement is an absolute, global guarantee that the current vertex cover S is genuinely a minimum vertex cover.
